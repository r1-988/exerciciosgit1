EXERCICIO 1- 
VOCE TRABALHA NUMA EQUIPE QUE DESEJA UTILIZAR O GIT EM SEUS PROJETOS.
VOCES POSSUEM UM REPOSITORIO CRIADO E PRECISAM CONFIGURAR PARA COMEÇAREM
A TRABALHAR.
* DESCREVA O QUE DEVERÁ SER FEITO PARA CONSEGUIR REALIZAR O CENARIO ACIMA.

R: LEVANDO EM CONSIDERAÇÃO QIE O REPOSITORIO FOI CRIADO E PRECISA SER COMPARTILHADO COM A EQUIPE, É
NECESSÁRIO CONVIDAR OS MEMBROS DA EQUEIPE ATRAVÉS DO NOME DO USUÁRIO. 
PARA QUE O PROJETO SEJA ACESSADO REMOTAMENTE, O USUÁRIO PRECISA CLONAR O REPOSITÓRIO LOCAL.
NESSE CASO, O USUÁRIO O COMANDO 'PULL', PUXANDO AS ULTIMAS ALTERAÇÕES DO REPOSITORIO.
ISSO GARANTE A COMPATIBILIDADE.

EXERCICIO 2-
VOCE E SEUS COLEGAS DE EQUIPE ESTÃO TRABALHANDO EM CUIMA DO MESMO CONJUNTO DE ARQUIVOS.
VOCE PRECSA DE UMA ATUALIZAÇÃO NO SEU REPOSITORIO QUE ALGUM DE SEUS COLEGAS ACABOU DE FINALIZAR E
FEZ A MERGE PARA A MAIN BRANCH.

R: NESSA SITUAÇÃO É NECESSÁRIO USAR O COMANDO 'CHECKOUT', EM SEGUIDA O COMANDO 'PULL'. 
RETORNAMOS A NOSSA BRANCH E EXECUTAMOS O 'REBASE'.

EXERCICIO 3-
APÓS UM DIA DE TRABALHO, VC FINALIZOU A CODIFICAÇÃO E PRECISA UNIR O SEU CODIGO COM A BRANCH MAIN.

R: É NECESSÁRIO EFETUAR O MERGE E DEPOIS INSERIR A TAG QUE INDICA QUE A CODIFICAÇÃO FINALIZOU.
EM SEGUIDA, FAREMOS UM 'PUSH' LEVANDO A TAG PARA A MAIN BRRANCH, SENDO DISPONIBILIZADA PARA PS DEMAIS USUARIOS.

EXERCICIO 5- APOS FINALIZAR UM PROBLEMA NO CODIGO, VOCÊ FEZ UMA PROPOSTA DO QUE PRECISA SER ALTERADO
PARA RESOLVER O BUG. PARA REALIZAR A CORREÇÃO, SERÁ NECESSARIO CRIAR UMA BRANCH, CODIFICAR A CORREÇÃO E
ENVIAR O CODIGO PARA QUE POSSA SER ANALISADO. 

R: COMANDO 'CHECKOUT' REALIZA-RÁ AS ATUALIZAÇÕES.
COMANDO 'PUSH' INSERE AS ALTERAÇÕES NA MAIN.
COMANDO 'TAG' INFORMA AS ALTERAÇÕES REALIZADAS.
COMANDO 'CLONE' PARA ACESSAR REMOTAMENTE.
COMANDO 'PULL' PARA ACESSAR AS ULTIMAS ATUALIZAÇÕES.
